# Homepage Viewport Rules

BUILD-039 establishes a practical review rule for the public homepage:

> A visitor should be able to understand each major homepage section within one viewport on desktop, iPad, and iPhone.

This does not mean every pixel of every section must fit on every device. It means the section's title, visual purpose, and primary action should be visible together without forcing the user to scroll through oversized internal panels.

## Rules

1. **One idea per section.** If a section contains multiple ideas, move the secondary ideas behind links, cards, or a deeper page.
2. **The visual and the heading must appear together.** A user should not see only the title of a section while the actual content sits below the fold.
3. **Short public copy.** Public homepage copy should usually be one short paragraph, not a documentation block.
4. **Cards should be compact.** Large card grids should show a preview and link to a full page.
5. **iPad is a primary review device.** If a section feels cut off on iPad, it needs layout repair.
6. **Advanced controls belong deeper.** Public homepage sections should demonstrate; research workspaces can expose full controls.
