# Architecture Index

The Orchid Continuum architecture library should organize source documents, standards, and conceptual frameworks into stable categories.

## Planned Categories

| Category | Purpose |
| --- | --- |
| Constitution | Foundational principles, rights, responsibilities, and governance commitments. |
| Cognitive Constitution | Principles for cognition, agency, memory, self-modeling, and assisted reasoning. |
| OCOS | Operating system concepts, layers, services, and interfaces for Orchid Continuum. |
| Connectome | Graph, memory, relationship, and knowledge-network models. |
| Calyx Executive Function | Planning, prioritization, orchestration, and executive-control concepts. |
| Knowledge Objects | Object model for durable concepts, artifacts, claims, sources, and transformations. |
| Relationship Model | Human, AI, project, document, and institutional relationship patterns. |
| Agent Framework | Agent roles, responsibilities, tools, handoffs, and safety boundaries. |
| Scientific Integrity | Evidence standards, uncertainty handling, citation practices, and anti-overclaiming rules. |
| Development Standards | Engineering principles, build workflows, code review, testing, and release expectations. |
| Build Lifecycle | Build IDs, branch strategy, PR lifecycle, deployment state, and completion records. |
| Future Vision | Long-range product, research, and ecosystem direction. |

## Curation Notes

- Source documents should be preserved with provenance.
- Derived summaries should identify source material.
- Architecture decisions should link to the build or PR that introduced them.
- Inaccessible architecture repositories should be recorded as workspace warnings until resolved.
